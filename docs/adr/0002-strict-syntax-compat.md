# 0002: 严格兼容三家原生语法

- 状态：Accepted
- 日期：2026-04
- 关联：`plan.md` §1、§4

## 背景
对图表渲染库而言，最大的迁移成本是语法。如果用户不能直接复制官方文档的示例跑通，就失去价值。

## 选项
- A：严格兼容，能直接跑官方 .mmd / .puml / .dot。
- B：兼容核心子集，边角语法不保证。
- C：另起一套统一的 Kotlin DSL。

## 决定
选 A。

## 影响
- 每个图类型必须配黄金语料（来自官方文档/样例库），作为回归测试。
- 解析器累积式诊断，不允许遇到不认识的语法直接崩溃。
- 工作量显著大于 B，反映在 Phase 4-6 的拆分粒度上。
